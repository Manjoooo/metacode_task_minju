# 전통 카탈로그의 4가지 한계

## 1. 파티션 리스트 부담 증가

전통적인 카탈로그 방식에서는 테이블의 파티션 정보를 메타스토어가 관리한다. 
데이터가 계속 쌓이고 파티션 수가 많아지면, 쿼리를 실행하기 전에 어떤 파티션을 읽어야 하는지 확인하는 과정 자체가 무거워질 수 있다.
즉, 실제 데이터 처리보다 파티션 목록을 조회하고 관리하는 과정이 병목이 될 수 있다. 
이처럼 파티션 수가 많아질수록 카탈로그 관리와 쿼리 성능에 부담이 커지는 문제가 발생한다.

## 2. 파티셔닝 변경의 어려움

전통적인 테이블 구조에서는 처음 정한 파티션 기준을 나중에 바꾸기 어렵다. 
예를 들어 처음에는 특정 기준으로 데이터를 나누었지만, 이후 분석 요구사항이 바뀌면 다른 기준의 파티셔닝이 필요할 수 있다.
하지만 기존 방식에서는 파티션 구조를 변경하려면 데이터를 다시 구성하거나 테이블을 새로 만드는 작업이 필요할 수 있다. 
따라서 분석 요구사항 변화에 유연하게 대응하기 어렵다는 한계가 있다.

## 3. 동시 쓰기 문제와 데이터 유실 가능성

전통적인 파일 기반 테이블에서는 여러 작업이 동시에 같은 테이블에 데이터를 쓰는 상황을 안정적으로 처리하기 어렵다. 
동시 쓰기 과정에서 하나의 작업 결과가 다른 작업 결과를 덮어쓰거나, 일부 파일이 누락될 가능성이 있다. 즉, 테이블 단위의 트랜잭션 보장이 약하면 데이터의 일관성이 깨질 수 있다. 
이 문제는 대규모 데이터 파이프라인에서 신뢰성 있는 테이블 운영을 어렵게 만든다.

## 4. 롤백 불가

전통적인 방식에서는 테이블의 과거 상태를 명확하게 관리하기 어렵다. 따라서 잘못된 데이터가 적재되었거나 문제가 있는 작업이 실행되었을 때, 이전의 정상 상태로 되돌리는 것이 쉽지 않다.
운영 환경에서는 데이터 오류가 발생했을 때 빠르게 복구할 수 있어야 한다. 
하지만 롤백 기능이 없다면 문제 발생 시 복구와 검증이 어려워진다.
